# Wanna add some charts?

# Starting from 2018-05-23, you will have to have at least 10 charts in the drive you're submitting.
# They will have to abide by decent charting standards: good beatmapping is a must, note placement and accuracy
# must be backed up by at least some musical logic. It's fine if you're not pitch perfect, we all have room for
# improvement: the requirements to get approved in here are lower than for the "charter role" on the CH Discord.

# If you feel like you apply to these guidelines, either submit a pull request by editing this file with
# a GitHub account or feel free to DM me on Discord (Paturages#9405) or a "chart reviewer" on the CH Discord
# server (type !charter in #charting for an up-to-date list of reviewers) with your drive link and how you wanna name it.

# Having the "charter role" on the CH Discord basically guarantees you getting on this list, so do mention that
# in the commit message when you're submitting, it will definitely help.

# Keep in mind that by editing this file through GitHub, you're not directly editing it,
# you are submitting a change that will have to go through my approval. 

# --------------------------------- #

# Source file guidelines
# (When in doubt, just follow the other examples, they work for a reason!)

# The format matches the following fields:
# <drive_name> :: <link> [:: <custom_script>]

# <drive_name> will show up on chorus as the label for the source link, along with the name of the subfolder it's in.
# It is searchable, so choose accordingly. Obviously, don't put "::" in it. I added the second ":"
# to enable people to use ":" in drive names, so don't ruin my fun!

# <link> is the link to your drive.
# Make sure to make your drive *public* first!
# The guidelines for folder/archive naming is gone: chorus now reads from your song.ini and .chart files.
# Make sure you fill them in properly: not only for chorus, but for Clone Hero as well!

# Optional: <custom_script> is for custom handlers. Default script is Google Drive (google-drive).
# (right now, you shouldn't have to use that at all in 99% of cases)

# proxy/<custom_script>:<url> is particularly useful if you have a front page you want to show
# instead of your drive folder (i.e. spreadsheet, website...).

# Current available custom scripts are:
# - google-drive: The default one for Google Drive
# - caddy: if you are using https://caddyserver.com/docs/browse to serve your files

# Lines that start with a # are comments and will not be parsed. Neither will empty lines.

# --------------------------------- #
